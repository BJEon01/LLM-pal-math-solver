# Validation Design

## 실험용 검증기 (GSM8K + PAL 성능 비교)

- 본 실험에서는 PAL이 생성한 Python 코드를 `Syntax`, `Security`, `Semantic`, `Runtime`의 4단계로 검증한다.
- `Syntax Validation`은 코드가 유효한 Python 문법인지, 그리고 산술 풀이용 코드 형태로 파싱 가능한지를 확인한다.
- `Security Validation`은 `import`, 파일 접근, 네트워크 접근, 동적 실행과 같은 위험한 구문을 AST 화이트리스트 방식으로 차단한다.
- `Semantic Validation`은 변수 정의 여부, 숫자형 연산 일관성, 최종 `answer` 변수의 존재와 수치형 여부를 검사한다.
- `Runtime Constraints`는 코드를 독립된 subprocess에서 실행하고, timeout을 통해 무한 루프 및 비정상 실행을 차단한다.
- 이를 통해 PAL 적용 여부뿐 아니라, 어떤 단계에서 코드가 실패하는지도 함께 분석할 수 있다.

## 배포용 확장 방향

- 실제 서비스에서는 사용자가 다양한 수학 문제를 입력하므로, GSM8K 실험용 검증기보다 더 넓은 처리 전략이 필요하다.
- 따라서 배포 단계에서는 문제 유형 분류를 먼저 수행하고, 산술 서술형, 대수식, 단위 변환 문제를 각각 별도의 처리기로 라우팅하는 구조를 목표로 한다.
- 또한 실행 환경은 프로세스 격리, 시간 제한, 자원 제한을 포함하는 강화된 샌드박스 구조로 확장할 계획이다.
